# 前台網站需求

## 網站性質
這是一個「作品目錄網站」。

## 驗證機制
網站有兩道驗證：
1. 成年驗證
2. 密碼驗證

## 解鎖前不可顯示作品
在這兩道驗證都尚未完成前：
- 不顯示任何作品
- 不顯示任何作品縮圖
- 不顯示任何作品占位框
- 不顯示任何作品數量
- 不顯示任何作品標題、價格、分類、標籤
- 不顯示任何能推測作品內容的版面

也就是說，解鎖前只會看到 gate 頁面，不會看到被遮罩的作品站。

## 允許預先下載部分資源
低畫質 WebP 預覽圖可以在未驗證時預先下載與快取，只要前端在成年驗證與密碼驗證完成前，不渲染任何作品內容、圖片或占位區塊即可。

# 解鎖後的作品目錄 UI

每張作品卡片只顯示 4 個資訊：
1. 作品名
2. 縮圖
3. 價格
4. 愛心

## 愛心功能
愛心是「本地收藏」，不是公開按讚數，不需要帳號系統。
行為：
- 點愛心加入收藏
- 再點一次取消收藏
- 收藏資料存在本機
- 右上角或右下角有一個固定的收藏清單入口
- 點愛心加入收藏時，要有飛行動畫，飛進角落的收藏清單入口
- 收藏清單可先做成側邊抽屜或簡單面板，優先簡單穩定

## 點作品縮圖
點作品縮圖後，不跳新頁，而是在原本目錄裡展開詳細內容。

展開後顯示：
1. 放大的主圖
2. 子縮圖列表
3. 內文

建議互動：
- 點卡片縮圖：展開/收合
- 點子縮圖：切換主圖
- 點愛心：只做收藏，不要觸發展開

# 圖片策略

## 圖片分級
### 1. 快速預覽圖
- 低畫質 WebP
- 明文
- 可預先載入
- 可快取
- 驗證前不可顯示

### 2. 清楚縮圖
- WebP 原始檔存在伺服器目錄
- 不可直接公開裸露給前端
- 由 Go 在請求時加密後回傳前端
- 前端使用密碼導出金鑰後本地解密顯示

### 3. 大圖
- WebP 原始檔存在伺服器目錄
- 不可直接公開裸露給前端
- 由 Go 在請求時加密後回傳前端
- 前端本地解密顯示

## 使用者體驗
解鎖後：
- 卡片先顯示低畫質預覽圖
- 再替換成清楚縮圖
- 展開時先用已有圖，再載入更清楚內容
- 主圖切換要順暢，避免閃爍
- 可做模糊到清晰的過渡

# Service Worker 與離線策略

這個網站必須有 Service Worker。

目標：
1. 主網域正常時，正常顯示網站並更新快取
2. 主網域無法連接時，顯示網站完整備份
3. 背景從備援域名列表嘗試更新網站
4. 使用者永遠只使用主網址，不直接看到備援網址

## 備援更新方向
- 主站掛掉時，Service Worker 要能用快取顯示完整網站
- 並且依序嘗試備援域名拉取更新
- 抓到後寫回主站自己的 cache
- 下次使用者仍是用主網址看到更新內容

# 備援域名分發與更新策略

這部分是核心需求，請不要忽略。

## 基本目標
備援域名列表不能對所有用戶都完全相同。
需要依照不同用戶「隨機發放不同的備援域名子集合」，降低所有用戶持有同一份完整備援名單而被一次性集中封鎖的風險。

## 分發原則
- 後端維護一個較大的備援域名池
- 每個用戶 / 每個客戶端初次取得備援列表時，只拿到其中一部分，不是完整池
- 不同用戶拿到的備援列表應儘量不同，但仍需保有一定重疊以避免完全碎片化
- 這個分發結果需要有穩定性，不要每次載入都隨機重洗
- 可依據某個客戶端識別、安裝識別、首次生成的本地 id、或其他穩定方式來派發
- 請設計成不依賴登入帳號也可運作

## 更新原則
備援列表一旦分發給某個用戶後，不要頻繁整包更新。
請採用「最小更新策略」：
- 只替換已失效的備援域名
- 仍然可用的備援域名要盡量保留
- 不要因為後端有新池就把整份列表全部洗掉
- 避免讓大量用戶在短時間內收到相同新名單，增加被一次封鎖的風險

## 請設計
1. 備援域名池的資料模型
2. 單一用戶已分配備援列表的本地持久化方式
3. 失效判定方式
4. 失效後如何補發新的備援域名
5. 補發時如何避免大量用戶補到完全相同的新域名
6. Service Worker / 前端 / 後端之間如何協作
7. 後台如何管理整個備援池與失效標記

## 我要的效果
- 每位用戶拿到的是自己的備援子集合
- 這個集合相對穩定
- 只在某些備援真的失效時，才局部替換
- 替換也盡量分散，不要大規模同步漂移到同一批新域名

# lazy load / 優先載入 / 預載需求

請實作清楚的資源載入分級。

## 載入優先順序
### 第 0 級
gate 頁本身需要的資源

### 第 1 級
解鎖後，視覺區內卡片的低畫質預覽圖

### 第 2 級
視覺區內卡片的清楚縮圖（加密後傳輸）

### 第 3 級
視覺區外的預覽圖、清楚縮圖，以及展開後可能需要的大圖預載

## 原則
- 先完成視覺區
- 視覺區完成後再預抓視外區
- 使用 Intersection Observer 或其他穩定方案
- 不要讓全部圖片同時搶載入
- 要考慮低畫質預覽圖、清楚縮圖、大圖三層資源的調度
